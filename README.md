# 模型回复自动评测系统

对大模型的多组回复进行自动两两对比评测。上传包含多个模型回复的 Excel 文件，系统自动识别回复列、生成全排列组合、调用 LLM 逐组评估，并输出带排名和分析的结果，生成文件后可供下载。

## 功能特性

- **自动列识别**：自动匹配 `回复1`、`回复2`、`回复3` 等列名，支持任意数量的回复列
- **全排列对比**：自动生成所有两两组合（3个回复列3组对比，4个回复列6组对比）
- **自定义提示词**：支持直接粘贴文本或上传 Excel 文件（需包含prompt列）
- **并发加速**：可配置并发数（1-20），多线程并行调用 LLM
- **实时进度**：前端实时展示评测进度条、状态消息
- **一键下载**：评测完成后直接下载结果 Excel
- **多模型兼容**：兼容所有 OpenAI 接口协议（DashScope、DeepSeek 等）


## 快速开始

### 1. 安装依赖

```bash
pip install flask pandas openpyxl openai
pip3 install -r requirements.txt

```

### 2. 准备前端文件

```bash
mkdir -p templates
cp "index.html" templates/index.html
```

### 3. 启动服务

```bash
python3 app.py
```

启动后访问 `http://127.0.0.1:5000/`

### 4. 云平台部署

项目已包含 `Procfile`，可直接部署到 Heroku 等支持 Procfile 的平台：

```
web: python app.py
```

## 数据文件格式

上传的 Excel 文件（`.xlsx`）需包含以下列：

| 列名 | 说明 | 是否必填 |
|------|------|---------|
| `System` | System Prompt 内容 | 选填 |
| `对话历史` | 对话历史文本 | 选填 |
| `回复1` | 第一个模型的回复 | **必填** |
| `回复2` | 第二个模型的回复 | **必填** |
| `回复3` | 第三个模型的回复（可选） | 选填 |

> 回复列必须命名为 `回复1`、`回复2`、`回复3`... 的格式，系统自动识别并生成所有两两组合。

## 提示词模板

提示词中可使用以下占位符，系统自动替换为实际数据：

| 占位符 | 替换内容 |
|--------|---------|
| `{{system}}` | System Prompt |
| `{{chat}}` | 对话历史 |
| `{{bot1}}` | 回复 A |
| `{{bot2}}` | 回复 B |

**示例：**

```
请对比以下两个模型回复的质量：

【System Prompt】{{system}}
【对话历史】{{chat}}
【回复A】{{bot1}}
【回复B】{{bot2}}

请从准确性、完整性、安全性三个维度对比，输出 JSON：
{"rank": "A>B", "analysis": "分析内容"}
```

## 输出结果

评测完成后的 Excel 在原始数据基础上新增列：

| 新增列 | 说明 |
|--------|------|
| `1vs2_rank` | 回复1 vs 回复2 的排名（如 A>B、A<B、A=B） |
| `1vs2_analysis` | 回复1 vs 回复2 的分析说明 |
| `1vs3_rank` | 回复1 vs 回复3 的排名 |
| `1vs3_analysis` | 回复1 vs 回复3 的分析说明 |
| ... | 所有组合的排名和分析列 |

## 配置说明

| 配置项 | 说明 | 默认值 |
|--------|------|--------|
| `PORT` 环境变量 | 服务端口 | 5000 |
| `MAX_CONTENT_LENGTH` | 上传文件大小限制 | 50 MB |
| `temperature` | LLM 评测温度 | 0.0（确保一致性） |
| `max_tokens` | LLM 最大输出 | 8192 |
| `max_retries` | 单次评测失败重试次数 | 3 |
